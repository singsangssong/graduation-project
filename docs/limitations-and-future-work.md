# 현재 한계와 향후 확장

## 구현 범위

Agentic Middleware는 다중 에이전트의 공유 자원 접근을 설명하고 검증하기 위한 연구
프로토타입입니다. 관련 연구의 개념을 하나의 실행 흐름으로 연결하지만, 각 연구의 전체
시스템이나 알고리즘을 재현하지는 않습니다.

| 관련 개념 | 현재 적용 범위 | 포함하지 않은 범위 |
| --- | --- | --- |
| AIOS | 에이전트와 자원 서비스 사이의 관리 계층 | 전체 에이전트 OS와 LLM 스케줄러 |
| SagaLLM | 체크포인트, 검증, 보상, 복구 수명주기 | 전체 계획 프레임워크와 독립 LLM 검증 에이전트 |
| ATCC | 비용 신호 기반 커밋 후보 중재 | 강화학습 기반 적응형 동시성 제어 |
| QCFuse | 질의 문맥 기반 동시 자원 읽기 융합 | KV 캐시 융합과 selective token recomputation |

## 현재 한계

### 단일 노드 SQLite

현재 저장소는 SQLite와 단일 DB 연결을 사용합니다. 구현과 재현이 단순하고 Saga
영속성을 확인하기에는 적합하지만, 다중 노드 환경의 락 경합이나 장애 조치를 설명하지
못합니다.

### 제한된 자원 모델

내장 실행기는 `reserve <resource_id>`와 `release <resource_id>` 행동만 직접
처리합니다. 실제 서비스의 결제, 배송, 좌석 배정과 같은 효과는 각각 다른 보상 규칙과
실패 정책이 필요합니다.

### 결정적 검증

현재 `ValidateSaga`는 완료된 체크포인트 존재 여부와 단계 상태를 확인합니다. 작업
결과의 의미적 정확성이나 LLM 응답 품질을 검증하지 않습니다.

### 단순화된 비용 점수

토큰 수와 추론 지연시간의 고정 가중치 합을 사용합니다. 보고된 값의 신뢰성, 사용자
우선순위, 성공 가능성, 작업의 실제 가치와 공정성은 반영하지 않습니다.

### 문자열 기반 읽기 융합

`resource_id + intent`가 동일한 요청만 융합합니다. 의미적으로 유사하지만 문자열이
다른 요청은 별도로 처리되며, 결과 버전이나 유효 기간도 확인하지 않습니다.

### 통제된 실험 환경

실험은 동일 자원에 요청이 집중되는 ticket-stock 시뮬레이션입니다. 논리적 조회
카운터, 고정된 비용 입력과 단일 머신 환경을 사용하므로 실제 운영 시스템으로 결과를
일반화할 수 없습니다.

## 향후 확장

### PostgreSQL 및 실제 락 경합 측정

SQLite 저장소를 PostgreSQL로 교체하고 격리 수준, lock wait time, deadlock,
advisory lock을 측정하면 실제 DB 동시성 제어와의 관계를 분석할 수 있습니다.

### 적응형 중재 정책

요청의 실행 시간, 접근 패턴, 충돌 빈도와 롤백 비용을 관찰해 낙관적·비관적 정책을
동적으로 선택할 수 있습니다. 공정성과 starvation 방지를 포함한 다목적 정책도
필요합니다.

### 외부 효과 보상 레지스트리

보상 행동을 typed handler registry로 분리하고 외부 API별 idempotency key,
재시도 정책, timeout과 dead-letter 처리를 추가할 수 있습니다.

### 의미 기반 읽기 융합

intent embedding이나 도메인 규칙을 사용해 의미적으로 호환 가능한 읽기를 찾을 수
있습니다. 이 경우 잘못된 융합을 막기 위한 결과 버전과 일관성 조건이 함께 필요합니다.

### 독립 검증 계층

결정적 검증 뒤에 선택적 LLM validator를 연결하고, 검증 정확도와 추가 비용을 함께
측정할 수 있습니다. LLM 검증 실패 시 재시도와 사람 승인 경로도 고려해야 합니다.

### 분산 실행과 관찰 가능성

다중 미들웨어 인스턴스를 위한 리더 선출 또는 분산 중재, OpenTelemetry trace,
구조화 로그와 장기 metrics 저장을 추가하면 운영 환경에서의 분석 가능성을 높일 수
있습니다.
